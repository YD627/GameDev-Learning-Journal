# 2026-05-01 至 2026-05-06 学习记录：C++面向对象核心与图形学变换系统

## 整体学习概要
在5月1日至5月6日期间，我的学习重点从图形学编程实践转向了**C++面向对象编程的核心机制**与**图形学变换系统的理论基础**。这段时期系统性地掌握了C++的封装特性、对象生命周期管理、内存模型，并深入理解了3D图形学中的坐标变换系统。这为构建更复杂的图形应用程序和游戏引擎奠定了坚实的语言基础和理论框架。

## 一、C++面向对象编程深度掌握

### 1. 封装(Encapsulation)与访问控制
*   **封装的意义**：理解了将数据（属性）和操作数据的方法（行为）捆绑在一起，形成“类”这一核心面向对象概念。掌握了通过访问权限控制（`public`、`private`、`protected`）来隐藏内部实现细节，仅暴露必要接口，提高代码的安全性和可维护性。
*   **`class`与`struct`的关键区别**：明确了在C++中两者唯一的本质区别在于默认的访问权限（`class`默认为`private`，`struct`默认为`public`），这是进行类设计时的基础选择依据。

### 2. 对象的初始化和清理
*   **构造函数与析构函数**：掌握了这两个特殊成员函数的语法、调用时机与核心作用。构造函数用于对象创建时的初始化，析构函数用于对象销毁时的资源清理，均由编译器自动调用，确保了对象的生命周期管理。
*   **构造函数的分类与调用**：学习了按参数（有参/无参）和类型（普通/拷贝）的分类方式，以及直接调用、拷贝调用和隐式转换三种调用方法。
*   **拷贝构造函数**：深入理解了拷贝构造函数的调用时机（初始化新对象、值传递、值返回），并掌握了编译器提供的默认拷贝构造规则。
*   **深拷贝与浅拷贝**：明确了这是处理类中含有动态分配资源（如指针）时的核心概念。理解了浅拷贝只复制指针地址导致的“双重删除”、“数据意外修改”等问题，以及深拷贝通过重新分配内存来复制完整数据的必要性。
*   **初始化列表**：学习了使用初始化列表语法来初始化类成员变量，这是一种更高效且必需的初始化方式（特别是对于`const`成员和引用成员）。
*   **类对象作为类成员**：理解了当类中包含其他类对象作为成员时，构造与析构的调用顺序（先成员对象，后自身；析构顺序相反）。
*   **静态成员**：掌握了静态成员变量（类内声明、类外初始化，所有对象共享）和静态成员函数（只能访问静态成员）的特性与用法。

### 3. C++对象模型与`this`指针
*   **成员存储模型**：认识到只有非静态成员变量属于类的对象实例，存储在对象内存中；静态成员变量、静态和非静态成员函数均独立于对象存储。
*   **`this`指针的本质与用途**：理解了`this`指针是一个隐含在每一个非静态成员函数中的常量指针，指向调用该函数的对象实例。掌握了其核心用途：区分同名形参与成员变量，以及在成员函数中返回对象自身（`return *this`）以支持链式调用。
*   **空指针访问成员函数**：了解了空指针可以调用不涉及`this`指针的成员函数，但对于使用了`this`的函数，必须进行判空保护以确保程序健壮性。
*   **`const`修饰成员函数**：掌握了常函数（函数后加`const`）和常对象的概念。明确了常函数内不能修改普通成员属性（`mutable`修饰的除外），而常对象只能调用常函数。

### 4. 内存池(Memory Pool)技术
*   **默认内存管理的不足**：理解了频繁使用`new`/`delete`或`malloc()`/`free()`可能导致性能下降（查找空闲块、切割、合并、加锁）和内存碎片问题。
*   **内存池的原理与优势**：掌握了内存池通过预先申请并管理一批大小固定的内存块，在程序运行时进行快速分配和释放，从而提升内存分配效率、减少碎片的思想。
*   **经典内存池实现**：通过学习一个模板化的内存池示例代码，理解了其核心设计：维护一个**内存块链表**（`MemBlock`）来管理从系统申请的大块内存，以及一个**空闲节点链表**（`FreeNode`）来快速分配和回收固定大小的对象单元。掌握了`malloc`和`free`操作在链表上的实现逻辑。

## 二、图形学变换与坐标系统理论深化

### 1. 变换(Transform)的数学基础与GLM应用
*   **变换的必要性**：认识到使用矩阵进行变换是高效实现物体移动、旋转、缩放的关键，优于逐帧修改顶点数据。
*   **GLM库的使用**：学习了如何使用GLM（OpenGL Mathematics）库来创建和操作变换矩阵（平移`glm::translate`、旋转`glm::rotate`、缩放`glm::scale`）。
*   **矩阵传递到着色器**：掌握了在C++端计算好变换矩阵后，如何通过`glGetUniformLocation`查询`uniform`变量地址，并使用`glUniformMatrix4fv`函数将矩阵数据传递到顶点着色器中，与顶点位置相乘以实现变换。

### 2. 坐标系统(Coordinate System)全流程
*   **坐标变换管线**：系统掌握了顶点从局部坐标到最终屏幕坐标的完整变换序列：**局部空间 -> （模型变换） -> 世界空间 -> （观察变换） -> 观察空间 -> （投影变换） -> 裁剪空间 -> （透视除法） -> 标准化设备坐标 -> （视口变换） -> 屏幕坐标**。
*   **核心变换矩阵**：
    *   **模型矩阵(Model Matrix)**：将物体从局部空间定位到世界空间，包含位移、旋转、缩放。
    *   **观察矩阵(View Matrix)**：将世界空间中的所有物体变换到摄像机视角下的观察空间。
    *   **投影矩阵(Projection Matrix)**：定义了观察空间到裁剪空间的映射，有两种类型：
        *   **正射投影(Orthographic Projection)**：使用`glm::ortho`创建，无透视效果，用于2D渲染或某些UI。
        *   **透视投影(Perspective Projection)**：使用`glm::perspective`创建，模拟人眼视觉效果，有“近大远小”的透视感。
*   **组合变换与传递**：理解了在着色器中组合这些矩阵的顺序应为 `投影矩阵 * 观察矩阵 * 模型矩阵 * 顶点坐标`（从右向左应用）。学习了在渲染循环中动态更新并传递这些矩阵以实现3D动画（如旋转的立方体）。
*   **深度测试(Z-buffering)**：掌握了启用深度测试（`glEnable(GL_DEPTH_TEST)`）的重要性，它通过深度缓冲（Z-buffer）自动处理片段的前后遮挡关系，并在每帧渲染前清除深度缓冲（`GL_DEPTH_BUFFER_BIT`）。
