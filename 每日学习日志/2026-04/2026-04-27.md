## 一、C++智能指针深入学习

### 1. 智能指针的必要性与发展历程
*   **核心价值**：自动管理动态分配的内存，**避免内存泄漏**，是现代C++资源管理的基石。
*   **演进历程**：从C++98的`auto_ptr`（已废弃）到C++11引入的`unique_ptr`、`shared_ptr`、`weak_ptr`，形成了完善的智能指针体系。

### 2. 四种智能指针的特性与用法对比
| 智能指针类型 | 所有权语义 | 特性 | 适用场景 | 注意事项 |
|-------------|-----------|------|---------|---------|
| **`auto_ptr`** (C++98) | 独占所有权 | 1. 复制/赋值转移所有权<br>2. 不支持容器<br>3. 不支持数组 | 历史代码兼容 | **C++11已废弃**，不建议在新项目中使用 |
| **`unique_ptr`** (C++11) | 独占所有权 | 1. 禁止拷贝，允许移动<br>2. 支持容器<br>3. 支持数组管理<br>4. 可指定自定义删除器 | 明确独占资源的场景<br>工厂函数返回值<br>数组内存管理 | 通过`std::move()`转移所有权 |
| **`shared_ptr`** (C++11) | 共享所有权 | 1. 基于引用计数<br>2. 支持拷贝和赋值<br>3. 支持`make_shared`高效构造<br>4. 可指定自定义删除器 | 多个对象共享同一资源<br>缓存、资源共享场景 | 警惕**循环引用**问题<br>避免原始指针交叉管理 |
| **`weak_ptr`** (C++11) | 弱引用观察 | 1. 不增加引用计数<br>2. 需通过`lock()`转为`shared_ptr`使用<br>3. 可检查`expired()`状态 | 打破`shared_ptr`循环引用<br>观察者模式、缓存 | 不能直接访问资源<br>需检查有效性后使用 |

### 3. 智能指针的使用陷阱与最佳实践
*   **常见陷阱**：
    1. 同一原始指针初始化多个智能指针
    2. 使用`get()`返回的指针初始化另一个智能指针
    3. 手动删除`get()`返回的指针
    4. 忽略`release()`的返回值导致内存泄漏
*   **循环引用解决方案**：使用`weak_ptr`替代`shared_ptr`建立弱引用关系
*   **性能考量**：`make_shared`比直接`new`更高效（单次内存分配）

## 二、OpenGL三角形绘制流程详解

### 1. 图形渲染管线（Graphics Pipeline）完整理解
掌握了从顶点数据到屏幕像素的七个核心阶段：
```
1. 顶点输入 -> 2. 顶点着色器 -> 3. 几何着色器 -> 4. 图元装配 -> 5. 光栅化 -> 6. 片段着色器 -> 7. 测试与混合
```

### 2. 核心缓冲区对象与配置流程
*   **标准化设备坐标（NDC）**：OpenGL处理的3D坐标范围（-1.0到1.0）
*   **顶点缓冲对象（VBO）**：
    - 作用：在GPU显存中存储顶点数据
    - 创建流程：`glGenBuffers()` -> `glBindBuffer()` -> `glBufferData()`
    - 使用提示：根据数据更新频率选择`GL_STATIC_DRAW`/`GL_DYNAMIC_DRAW`/`GL_STREAM_DRAW`
*   **顶点数组对象（VAO）**：
    - 作用：存储顶点属性配置，简化状态管理
    - 创建流程：`glGenVertexArrays()` -> `glBindVertexArray()`
*   **顶点属性配置**：
    ```cpp
    glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3*sizeof(float), (void*)0);
    glEnableVertexAttribArray(0);
    ```
    - 参数详解：位置、大小、类型、标准化、步长、偏移量

### 3. 着色器程序完整生命周期
*   **顶点着色器（Vertex Shader）**：
    ```glsl
    #version 330 core
    layout (location = 0) in vec3 aPos;
    void main() { gl_Position = vec4(aPos, 1.0); }
    ```
*   **片段着色器（Fragment Shader）**：
    ```glsl
    #version 330 core
    out vec4 FragColor;
    void main() { FragColor = vec4(1.0, 0.5, 0.2, 1.0); }
    ```
*   **着色器编译与链接流程**：
    1. 创建：`glCreateShader()` 
    2. 指定源码：`glShaderSource()`
    3. 编译：`glCompileShader()`，需检查编译状态
    4. 创建程序：`glCreateProgram()`
    5. 附加着色器：`glAttachShader()`
    6. 链接：`glLinkProgram()`，需检查链接状态
    7. 清理：`glDeleteShader()`
*   **绘制命令**：`glDrawArrays(GL_TRIANGLES, 0, 3)`

## 三、光栅化理论基础

### 1. 从规范坐标到屏幕坐标
*   **视口变换**：将NDC坐标`[-1,1]²`映射到屏幕坐标`[0,width]×[0,height]`
*   **视口变换矩阵**：
    ```
    M_viewport = [width/2,  0,        0,  width/2]
                 [0,        height/2, 0,  height/2]
                 [0,        0,        1,  0      ]
                 [0,        0,        0,  1      ]
    ```

### 2. 三角形光栅化算法
*   **采样法核心思想**：判断每个像素中心`(x+0.5, y+0.5)`是否在三角形内
*   **点在三角形内的判断**：通过三次叉积`(QA,AB)、(QB,BC)、(QC,CA)`方向一致性判断
*   **包围盒优化**：只在三角形AABB内进行采样测试，减少计算量
*   **走样（Aliasing）问题**：采样频率不足导致信号失真，表现为锯齿边缘

### 3. 抗锯齿技术
*   **走样根源**：信号频率高于采样频率的一半（违反奈奎斯特采样定理）
*   **预滤波（Pre-Filter）**：先模糊后采样，如使用1-pixel box-blur滤波器
*   **超采样抗锯齿（MSAA）**：
    - 原理：每个像素内进行多次子采样
    - 实现：像素划分为N个子样本，计算覆盖率，按覆盖率混合颜色
    - 权衡：质量提升 vs 性能开销
*   **快速近似抗锯齿（FXAA）**：后处理技术，识别并平滑图像中的高频边缘
*   **时间性抗锯齿（TAA）**：利用时间连续性，混合多帧信息提高有效采样率
