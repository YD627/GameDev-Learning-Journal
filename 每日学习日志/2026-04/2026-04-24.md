# 2026-04-24 学习记录：C++动态内存管理与图形学2D变换

今天并行学习了**C++中的动态内存管理机制**与**图形学基础的2D变换数学**，这两者分别是控制程序内存与描述图形空间操作的核心知识。

## 核心知识点总结

### 一、C++动态内存管理
1.  **管理方式对比**：深入理解了C++的`new`/`delete`运算符与C语言的`malloc`/`free`函数在本质、初始化行为、类型安全、异常处理以及对对象构造/析构函数调用上的关键区别。
2.  **内存泄漏**：明确了内存泄漏的定义、危害（资源浪费、性能下降、程序不稳定）及主要原因（如未正确配对使用`new[]`和`delete[]`），建立了主动防范内存泄漏的意识。
3.  **数组内存布局**：认识到动态分配的**多维数组其内存是否连续取决于分配方式**：
    *   使用`new int[3][4]`会获得连续的地址空间。
    *   先分配指针数组再分段分配行，会导致各行存储在堆中可能不连续的位置。
4.  **对象存储与生命周期**：
    *   通过`new`创建的对象存储在**堆(Heap)**，需手动管理。
    *   局部对象存储在**栈(Stack)**，由系统自动管理。
    *   对象数组的构造顺序（从低地址到高地址）与析构顺序（相反）的规律。
5.  **语法细节**：
    *   `new`可用于单对象初始化（如`new int(10)`）。
    *   结构体(`struct`)与类(`class`)在功能上几乎完全等价，核心语法区别仅在于**默认的成员访问权限和继承方式**。

### 二、图形学基础：2D变换
1.  **基本变换原理**：
    *   **缩放（Scale）**：通过缩放因子 `(s_x, s_y)` 改变点的坐标，属于线性变换：`x' = s_x * x`, `y' = s_y * y`。
    *   **旋转（Rotation）**：绕原点旋转角度 `θ`，通过旋转矩阵实现：
        ```
        R(θ) = [ cos(θ), -sin(θ) ]
               [ sin(θ),  cos(θ) ]
        ```
2.  **齐次坐标（Homogeneous Coordinates）的关键引入**：
    *   **解决的问题**：平移（Translation）变换 `(x' = x + t_x, y' = y + t_y)` 无法用2x2矩阵表示，因为它不是线性变换。
    *   **解决方案**：增加一个维度，将点 `[x, y]` 表示为 `[x, y, 1]`。平移、旋转、缩放因此可以统一用**3x3矩阵**来表示和组合。
    *   **统一表示**：一个同时包含旋转和平移的变换矩阵为：
        ```
        [ cos(θ), -sin(θ), t_x ]
        [ sin(θ),  cos(θ), t_y ]
        [   0,       0,     1  ]
        ```
    *   **核心优势**：使得复杂的变换可以通过**矩阵连乘**来方便地组合，是后续理解3D图形学中模型、视图、投影变换的基石。

## 知识联系与学习体会
今日的学习构成了一个有趣的对比：**C++内存管理关注的是数据在计算机物理内存中的“生存”方式**，而**图形学变换关注的是数据在虚拟空间中的“运动”与“关系”**。两者的结合点在于：当我们用C++程序实现图形学算法时，那些代表点、向量、变换矩阵的数据结构（如`Vec3`, `Mat4`），其生命周期需要通过动态内存来高效管理，其计算结果的正确性则依赖于深刻的数学理解。理解齐次坐标，是迈向用代码构建虚拟世界的第一步。
