2. 顶点输入
    - 定义：开始绘制图形之前，我们需要先给OpenGL输入一些顶点数据。OpenGL是一个3D图形库，所以在OpenGL中我们指定的所有坐标都是3D坐标（x、y和z）。OpenGL不是简单地把所有的3D坐标变换为屏幕上的2D像素；OpenGL仅当3D坐标在3个轴（x、y和z）上-1.0到1.0的范围内时才处理它。所有在这个范围内的坐标叫做标准化设备坐标(Normalized Device Coordinates)，此范围内的坐标最终显示在屏幕上（在这个范围以外的坐标则不会显示）。
    - 三角形顶点数据定义
        ```C++
        float vertices[] = {
            -0.5f, -0.5f, 0.0f,
             0.5f, -0.5f, 0.0f,
             0.0f,  0.5f, 0.0f
        };
        ```

3. VBO(Vertex Buffer Object，顶点缓冲区对象)
    - 定义：VBO是一种用于存储顶点数据的特殊缓冲区。VBO可以存储顶点数据、颜色数据、纹理坐标数据等。
    - 优势：它会在GPU内存（通常被称为显存）中储存大量顶点。使用这些缓冲对象的好处是我们可以一次性的发送一大批数据到显卡上，而不是每个顶点发送一次。从CPU把数据发送到显卡相对较慢，所以只要可能我们都要尝试尽量一次性发送尽可能多的数据。当数据发送至显卡的内存中后，顶点着色器几乎能立即访问顶点，这是个非常快的过程。
    - 创建VBO对象
        ```C++
        unsigned int VBO;
        glGenBuffers(1, &VBO);
        glBindBuffer(GL_ARRAY_BUFFER, VBO);
        glBufferData(GL_ARRAY_BUFFER, sizeof(vertices), vertices, GL_STATIC_DRAW);
        ```
        - glGenBuffers: 用来生成VBO对象。
        - glBindBuffer：把新创建的缓冲区绑定到GL_ARRAY_BUFFER目标。
        - glBufferData：是一个专门用来把用户定义的数据复制到当前绑定缓冲区的函数。
            - 第一个参数是缓冲区类型，这里我们使用GL_ARRAY_BUFFER。
            - 第二个参数是缓冲区的大小，这里我们使用sizeof(vertices)。
            - 第三个参数是缓冲区的数据，这里我们使用vertices数组。
            - 第四个参数是缓冲区的数据类型，有3种可能的值：GL_STATIC_DRAW，GL_DYNAMIC_DRAW和GL_STREAM_DRAW。这里我们使用GL_STATIC_DRAW。
                - GL_STATIC_DRAW：数据不会或几乎不会改变。
                - GL_DYNAMIC_DRAW：数据会被改变很多。
                - GL_STREAM_DRAW：数据每次绘制时都会改变。

4. 顶点着色器
    - 定义：顶点着色器是一个特殊类型的着色器，它负责处理顶点数据。
    - 顶点着色器的输入：顶点数据。
    - 顶点着色器的输出：顶点位置、法线、纹理坐标等。
    - 基础的GLSL代码：
        ```GLSL
        #version 330 core
        layout (location = 0) in vec3 aPos;
        void main() {
            gl_Position = vec4(aPos, 1.0);
        }
        ```
        - #version 330 core：指定OpenGL版本为3.3，使用核心配置文件。
        - layout (location = 0) in vec3 aPos：指定顶点数据的输入变量aPos的布局位置为0。
        - void main()：主函数，必须有，返回值为void。
        - gl_Position = vec4(aPos, 1.0)：把顶点数据aPos转换为齐次坐标，并赋值给gl_Position。

5. 片段着色器
    - 定义：片段着色器是一个特殊类型的着色器，它负责处理片段数据。
    - 片段着色器的输入：片段数据。
    - 片段着色器的输出：片段颜色。
    - 基础的GLSL代码：
        ```GLSL
        #version 330 core
        out vec4 FragColor;
        void main() {
            FragColor = vec4(1.0, 0.5, 0.2, 1.0);
        }
        ```
        - #version 330 core：指定OpenGL版本为3.3，使用核心配置文件。
        - out vec4 FragColor：指定片段颜色的输出变量FragColor。
        - void main()：主函数，必须有，返回值为void。
        - FragColor = vec4(1.0, 0.5, 0.2, 1.0)：把片段颜色FragColor赋为橙色。

6. 编译着色器
    ```C++
    unsigned int vertexShader;
    vertexShader = glCreateShader(GL_VERTEX_SHADER);
    glShaderSource(vertexShader, 1, &vertexShaderSource, NULL);
    glCompileShader(vertexShader);
    ```
    - glCreateShader：创建一个新的着色器。
    - glShaderSource：把着色器代码赋值给着色器。
    - glCompileShader：编译着色器。

7. 着色器程序
    ```C++
    unsigned int shaderProgram;
    shaderProgram = glCreateProgram();
    glAttachShader(shaderProgram, vertexShader);
    glAttachShader(shaderProgram, fragmentShader);
    glLinkProgram(shaderProgram);

    glDeleteShader(vertexShader);
    glDeleteShader(fragmentShader);
    ```
    - glCreateProgram：创建一个新的着色器程序。
    - glAttachShader：把顶点着色器和片段着色器附加到着色器程序上。
    - glLinkProgram：链接着色器程序。
    - glUseProgram：使用着色器程序。
    - glDeleteShader：删除着色器。
    - glDeleteProgram：删除着色器程序。

8. VAO(Vertex Array Object，顶点数组对象)
    - 定义：VAO可以像顶点缓冲对象那样被绑定，任何随后的顶点属性调用都会储存在这个VAO中。
    - 创建VAO对象
        ```C++
        unsigned int VAO;
        glGenVertexArrays(1, &VAO);
        glBindVertexArray(VAO);
        ```
        - glGenVertexArrays：用来生成VAO对象。
        - glBindVertexArray：绑定VAO，从绑定之后起，我们应该绑定和配置对应的VBO和属性指针，之后解绑VAO供之后使用。当我们打算绘制一个物体的时候，我们只要在绘制物体前简单地把VAO绑定到希望使用的设定上就行了。
    - 顶点属性指针：
        ```C++
        glVertexAttribPointer(0, 3, GL_FLOAT, GL_FALSE, 3 * sizeof(float), (void*)0);
        glEnableVertexAttribArray(0);
        ```
        - glVertexAttribPointer：指定顶点属性的指针。
            - 第一个参数指定我们要配置的顶点属性。还记得我们在顶点着色器中使用layout(location = 0)定义了position顶点属性的位置值(Location)吗？它可以把顶点属性的位置值设置为0。因为我们希望把数据传递到这一个顶点属性中，所以这里我们传入0。
            - 第二个参数指定顶点属性的大小。顶点属性是一个vec3，它由3个值组成，所以大小是3。
            - 第三个参数指定数据的类型，这里是GL_FLOAT(GLSL中vec*都是由浮点数值组成的)。
            - 下个参数定义我们是否希望数据被标准化(Normalize)。如果我们设置为GL_TRUE，所有数据都会被映射到0（对于有符号型signed数据是-1）到1之间。我们把它设置为GL_FALSE。
            - 第五个参数叫做步长(Stride)，它告诉我们在连续的顶点属性组之间的间隔。由于下个组位置数据在3个float之后，我们把步长设置为3 * sizeof(float)。要注意的是由于我们知道这个数组是紧密排列的（在两个顶点属性之间没有空隙）我们也可以设置为0来让OpenGL决定具体步长是多少（只有当数值是紧密排列时才可用）。一旦我们有更多的顶点属性，我们就必须更小心地定义每个顶点属性之间的间隔，我们在后面会看到更多的例子（译注: 这个参数的意思简单说就是从这个属性第二次出现的地方到整个数组0位置之间有多少字节）。
            - 最后一个参数的类型是void*，所以需要我们进行这个奇怪的强制类型转换。它表示位置数据在缓冲中起始位置的偏移量(Offset)。由于位置数据在数组的开头，所以这里是0。我们会在后面详细解释这个参数。

9. EBO(Element Buffer Object，元素缓冲对象)
    - 定义：EBO可以像顶点缓冲对象那样被绑定，任何随后的元素索引调用都会储存在这个EBO中。
    - 索引绘制
        ```C++
        unsigned int indices[3] = {0, 1, 2};
        ```
    - 创建EBO对象并绑定到GL_ELEMENT_BUFFER_BUFFER
        ```C++
        unsigned int EBO;
        glGenBuffers(1, &EBO);

        // 位于VBO之后，EBO必须绑定到GL_ELEMENT_BUFFER_BUFFER上
        glBindBuffer(GL_ELEMENT_BUFFER_BUFFER, EBO);
        glBufferData(GL_ELEMENT_BUFFER_BUFFER, sizeof(indices), indices, GL_STATIC_DRAW);
        ```
    - 绑制三角形
        ```C++
        glDrawElements(GL_TRIANGLES, 3, GL_UNSIGNED_INT, 0);
        ```
        - glDrawElements：绘制元素索引。
            - 第一个参数指定要绘制的元素类型。
            - 第二个参数指定要绘制的顶点数量。
            - 第三个参数是索引的类型。
            - 第四个参数我们可以指定EBO中的偏移量（或者传递一个索引数组，但是这是当你不在使用索引缓冲对象的时候），但是我们会在这里填写0。
