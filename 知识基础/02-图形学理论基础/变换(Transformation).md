1. 二维缩放运算：
    ```
    x缩放因子：s1
    y缩放因子：s2
    [x'] = [s1] * [x]
    [y'] = [s2] * [y]
    ```
    2. 二维旋转运算：
    ```
    旋转角度: θ
    R(θ) = [cos(θ), -sin(θ)]
           [sin(θ),  cos(θ)]
    [x'] = R(θ) * [x]
    [y'] = R(θ) * [y]
    ```
    3. 齐次坐标运算：
    ```
    旋转角度: θ
    平移距离: tx, ty
    
    [x']   [cos(θ) -sin(θ) tx] [x]
    [y'] = [sin(θ)  cos(θ) ty] [y]
    [1]    [  0       0     1] [1]

    加上缩放后：
    [x']   [a b tx] [x]
    [y'] = [c d ty] [y]
    [1]    [0 0  1] [1]

    先旋转后位移：
    矩阵方向是从右往左的，所以先旋转后位移的矩阵是：
               [x]   [x']
    T * R(θ) * [y] = [y']
               [1]   [1 ]
    ```

3. MVP变换：Model-View-Projection变换\
    1. 视图变换(View Transformation):
       1. 什么是视图变换：寻找一个好的角度去放置相机，使场景中的对象在相机中显示得更清晰、更真实。
       2. 初步定义相机：
        - 位置(Position)
        - 目向(Look-at Direction)
        - 上方(Up Vector)
       3. 经常把相机移动到原点:
        - 位置：原点(0, 0, 0)
        - 目向：-Z方向(0, 0, 1)
        - 上方：Y方向(0, 1, 0)
        - 然后转换物体到相机坐标系
       4. 变换相机位置通过M_View矩阵实现
        - 原先位置是e点, 上方向是t向量, 目向是g向量 
        - 先把e点移动到原点
        - 再把g向量旋转到-Z方向
        - 接着把t向量旋转到Y方向
        - 旋转(g × t)到X方向
    
    2. 投影变换(Projection Transformation):
       1. 什么是投影变换：将相机坐标系中的点映射到屏幕坐标系中，使它们在屏幕上显示得更清晰、更真实。(3D -> 2D)
       2. 分类:
        - 正交投影(Orthographic Projection)：
          - 保持物体的大小和比例，不改变其方向。
          - 例如：在2D游戏或模拟中，物体的大小和比例是固定的，不会改变。
          - 通常会将[l,r] * [b,t] * [n,f]的长方体映射到[-1,1] * [-1,1] * [-1,1]的正方体：
          ```
          T_orthographic = [1    0    0  -(r+l)/2]
                           [0    1    0  -(t+b)/2]
                           [0    0    1  -(n+f)/2]
          ```

### 视图变换矩阵运算流程

视图变换（View Transformation）的目标是将世界空间中的相机和所有物体，一同变换到一个标准的、便于后续投影的“相机坐标系”中。其核心矩阵 **M_view** 的推导遵循明确的几何和数学步骤。

#### 1. 定义与目标
- **输入**：相机在世界空间中的位置 **e** (eye position)、看向目标点（可衍生出看向方向 **g**）、以及一个向上的方向 **t** (up vector)。
- **标准相机坐标系**：
  - 相机位于原点 **(0, 0, 0)**
  - 看向 **-Z** 轴方向
  - 向上方向为 **Y** 轴方向
- **输出**：视图变换矩阵 **M_view**。对世界空间中的任意一点 **P**，其在相机空间中的坐标 **P'** 为：**P' = M_view * P**

#### 2. 运算流程与矩阵构造
视图变换可分解为两步：**平移** 和 **旋转**。

**a. 平移矩阵 T_view**
首先将相机位置 **e** 平移至原点。
```math
T_view = [1, 0, 0, -x_e]
         [0, 1, 0, -y_e]
         [0, 0, 1, -z_e]
         [0, 0, 0,   1 ]
```
该矩阵将点 **e** 变换到 **(0,0,0)**。

**b. 旋转矩阵 R_view**
然后将相机坐标系旋转对齐到标准坐标轴。这需要构建一个由相机局部坐标轴构成的**正交基**。
1.  **计算相机局部坐标轴**（单位化后）：
    - 看向方向：**g** （由相机位置 **e** 指向目标点，需单位化）
    - 右方向：**r = t × g** （叉积，顺序取决于坐标系左右手性，常见为 t × g）
    - 上方向：**u = g × r** （重新计算，以确保三个轴严格正交）
2.  **构造旋转矩阵**：
    我们的目标是将世界坐标系的轴 (**X, Y, Z**) 旋转到与相机轴 (**r, u, -g**) 对齐。这等价于**将相机轴旋转到世界轴**的逆变换。
    - 矩阵 **R** 的**每一列**是**相机坐标轴**在世界坐标系中的表示：
    ```math
    R = [x_r, x_u, x_-g]
        [y_u, y_u, y_-g]
        [z_r, z_u, z_-g]
    ```
    - 我们需要的是其逆变换 **R⁻¹**，它将世界坐标系中的点变换到相机坐标系。由于 **R** 是正交矩阵（其列向量是单位正交基），其逆等于其转置：
    ```math
    R_view = R-1 = Rᵀ = [ x_r,  y_r,  z_r, 0]
                        [ x_u,  y_u,  z_u, 0]
                        [-x_g, -y_g, -z_g, 0]
                        [0,    0,    0,    1]
    ```

**c. 组合成最终视图矩阵 M_view**
变换顺序是：先平移相机到原点，再旋转对齐。在矩阵乘法中，变换从右向左应用。
```math
M_view = R_view * T_view
```

#### 3. 核心要点总结
1.  **两步走**：视图变换 = 平移相机位置到原点 + 旋转相机朝向到标准轴。
2.  **基的构造**：旋转矩阵的核心是计算相机正交基 **r**, **u**, **g**。
3.  **逆与转置**：利用正交矩阵 **R⁻¹ = Rᵀ** 的性质，可以避免求逆运算，直接通过将基向量**按行放置**来构造 **R_view**。
4.  **矩阵乘法顺序**：最终变换矩阵是 **R_view * T_view**，表示先应用平移 **T_view**，再应用旋转 **R_view**。
