## 多态
### 1. 多态的基本概念
*多态是C++面向对象三大特性之一*\
多态分为两类:
- 静态多态: 函数重载 和 运算符重载属于静态多态，复用函数名
- 动态多态: 派生类和虚函数实现运行时多态
静态多态和动态多态区别：
- 静态多态的函数地址早绑定 - 编译阶段确定函数地址
- 动态多态的函数地址晚绑定 - 运行阶段确定函数地址

总结：
- 多态满足条件
    - 有继承关系
    - 子类重写父类中的虚函数
- 多态使用条件
    - 父类指针或引用指向子类对象
- 重写：函数返回值类型 函数名 参数列表 完全一致称为重写

多态的优点:
- 代码组织结构清晰
- 可读性强
- 利于前期和后期的扩展以及维护

### 2. 纯虚函数和抽象类
在多态中，通常父类中虚函数的实现是毫无意义的，主要都是调用子类重写的内容。因此可以将虚函数改为纯虚函数。\
纯虚函数语法：`virtual 返回值类型 函数名 （参数列表）= 0 ;`\
当类中有了纯虚函数，这个类也称为*抽象类*
*抽象类特点：*
- 无法实例化对象
- 子类必须重写抽象类中的纯虚函数，否则也属于抽象类

### 3. 虚析构和纯虚析构
多态使用时，如果子类中有属性开辟到堆区，那么父类指针在释放时无法调用到子类的析构代码\
解决方法：
- 父类中添加虚析构函数
- 子类中添加纯虚析构函数
虚析构和纯虚析构共性：
- 可以解决父类指针释放子类对象
- 都需要有具体的函数实现
虚析构和纯虚析构区别：
- 如果是纯虚析构，该类属于抽象类，无法实例化对象
虚析构语法：`virtual ~类名(){};`\
纯虚析构语法：`virtual ~类名()=0;`  `类名::~类名(){};`

虚函数中的两个关键字：
- override: ：保证在派生类中声明的重载函数，与基类的虚函数有相同的签名；
- final：阻止类的进一步派生 和 虚函数的进一步重写。

### 4. 虚函数之vptr（虚指针）和vtbl(虚表)
### **虚函数之vptr与vtbl工作机制详解**

vptr（虚表指针）和vtbl（虚函数表）是C++实现**动态多态**（运行时多态）的核心底层机制。以下是其详细的运作原理：

**1. 内存布局与创建时机**
当一个类声明了至少一个虚函数（包括继承来的）时，编译器会秘密地为这个类做两件事：
- **生成虚函数表**：编译器会为该类创建一个**唯一的、静态的**虚函数表。这个表本质上是一个函数指针数组，按顺序存放该类所有虚函数的入口地址。
- **添加虚表指针**：在编译阶段，编译器会在该类每个对象的**内存布局的最前面**（通常如此）添加一个隐藏的成员变量——`vptr`。这个指针在对象构造时，由构造函数中的“秘密代码”初始化为指向该类对应的虚函数表。

例如，对于以下类：
```cpp
class Base {
public:
    virtual void func1() { /* ... */ }
    virtual void func2() { /* ... */ }
    void nonVirtualFunc() { /* ... */ }
    int data;
};
```
一个`Base`对象在内存中的简化布局大致如下：
```
Base 对象
+------------------+
| vptr (指向Base的虚表) | -> 隐藏成员，由编译器添加
+------------------+
| int data         | -> 普通成员
+------------------+
```
对应的`Base`虚表结构：
```
Base的虚函数表 (vtbl for Base)
+------------------+
| &Base::func1     |
+------------------+
| &Base::func2     |
+------------------+
```

**2. 动态绑定（晚绑定）的过程**
当通过基类指针或引用调用虚函数时，编译器不会直接生成调用具体函数的代码，而是生成一段“间接寻址”代码。这正是“函数地址晚绑定”的体现。
```cpp
Base* obj = new Derived(); // 假设Derived重写了func1
obj->func1(); // 此行代码在运行时发生以下步骤
```
1.  从`obj`指向的对象内存首部取出`vptr`。
2.  通过这个`vptr`找到该对象实际类型（例如`Derived`）的虚函数表。
3.  在虚表中查找`func1`对应的槽位（位置在编译时已确定）。
4.  通过该槽位中存储的函数地址，跳转并执行真正的函数（`Derived::func1`）。

这个过程与通过“虚继承”的`vbptr`访问虚基类成员在间接性上类似，但**目的完全不同**：`vptr`用于动态决定**调用哪个函数**，而`vbptr`用于计算**数据成员在内存中的位置**。

**3. 继承体系下的虚表结构**
在继承中，虚表会被继承和扩展，这是实现多态的关键。
```cpp
class Derived : public Base {
public:
    virtual void func1() override { /* ... */ } // 重写
    virtual void func3() { /* ... */ } // 新的虚函数
};
```
- **Derived的虚表**：编译器会先生成一份Base虚表的副本，然后进行更新：
    1.  用`Derived::func1`的地址覆盖（重写）从Base继承来的`func1`槽位。
    2.  在表尾追加`Derived`新声明的虚函数`func3`的地址。
- **Derived对象的vptr**：在构造`Derived`对象时，其`vptr`被设置为指向`Derived`自己的虚表。

```
Derived 对象
+-------------------+
| vptr (指向Derived的虚表) |
+-------------------+
| Base::data        |
+-------------------+
| Derived自身数据    |
+-------------------+

Derived的虚函数表
+-------------------+
| &Derived::func1   |  // 被覆盖
+-------------------+
| &Base::func2      |  // 未重写，继承
+-------------------+
| &Derived::func3   |  // 新增
+-------------------+
```

**4. 多继承下的复杂性**
在多继承下，一个派生类会包含多个基类的子对象，因此也可能拥有**多个vptr**，指向**多个虚表**。
```cpp
class Base1 { virtual void f1(); };
class Base2 { virtual void f2(); };
class MI : public Base1, public Base2 {
    virtual void f1() override;
    virtual void f3();
};
```
`MI`对象将包含两个vptr，分别指向两个虚表片段：一个为`Base1`部分（包含被重写的`f1`和新增的`f3`的指针），另一个为`Base2`部分。这使得通过`Base2*`指针调用虚函数时，需要进行`this`指针的偏移调整。

**5. 与文档中其他知识点的关联**
- **虚析构函数**：正是由于vptr的存在，当`delete`一个基类指针时，才能通过该指针找到对象实际类型的虚表，进而调用到正确的派生类析构函数，最后再调用基类析构函数。如果析构函数非虚，则只会进行静态绑定，调用基类的析构函数，导致派生类部分资源泄漏。
- **`override`与`final`关键字**：
    - `override`：是给编译器和程序员的提示，确保意图是重写虚函数。它本身不改变vptr/vtbl机制，但能帮助避免因签名错误导致的意外隐藏而非重写。
    - `final`：禁止后续派生类重写某个虚函数，编译器可以据此进行“去虚拟化”优化，即将虚函数调用优化为直接调用，提升性能。

**总结**
vptr和vtbl机制是C++实现运行时多态的基石。它以**空间（每个对象一个指针，每个类一张虚表）换取时间（一次间接跳转）** 的方式，提供了极其灵活的多态行为。理解这一机制，有助于深入理解C++对象模型、多态的开销来源，并能在调试中解释对象的内存布局和行为。
