## 内存模型

高地址\
┌─────────────────┐\
│             内核空间                │ ← 操作系统内核\
├─────────────────┤\
│               栈(stack)               │ ← 局部变量、函数参数\
│                     ↓                     │   自动分配/释放\
│                     ↑                     │\
├─────────────────┤\
│                堆(heap)              │ ← 动态分配内存(malloc/new)\
│                                            │   手动管理\
├─────────────────┤\
│                  BSS段                │ ← 未初始化的全局/静态变量\
├─────────────────┤\
│            数据段(Data)           │ ← 已初始化的全局/静态变量\
├─────────────────┤\
│            代码段(Text)             │ ← 程序代码、常量字符串\
└─────────────────┘\
低地址

- 内核空间：存放的是与操作系统相关的代码，用户没有权限去操作，如果要操作，需要调用**系统调用函数**来进行操作
    - 系统调用函数：用户态程序调用内核态函数，用于请求操作系统提供服务(如文件操作、网络操作等)
- 栈区：其空间**自上而下**进行增长，存放函数调用的数据（函数参数、函数体中的局部变量、寄存器信息），函数调用结束后，对应的**栈帧**被回收
    - 栈帧：每个函数调用时，会创建一个栈帧，用于存储该函数的局部变量、函数参数、寄存器信息等
- 堆区：其空间**自下而上**进行增长，用户**动态进行动态申请**的内存就在堆区，用户需在代码中申请，也需要自己在代码中将申请的内存给释放掉，否则会导致**内存泄漏**
    - 内存泄漏：用户在申请内存后，没有及时释放，导致内存被占用，无法被其他程序使用
- BSS段(内存映射段)：存放的是**未初始化的全局/静态变量**，其空间在程序运行时会被操作系统自动分配
- 数据段(Data)：存放的是**已初始化的全局/静态变量**，其空间在程序运行时会被操作系统自动分配
- 代码段(Text)：放置的是**用户代码和字符串常量**，该位置的数据是不能修改的，只能读取
    - 代码段：程序运行时，会将代码段中的指令加载到内存中，然后执行
    - 字符串常量：程序运行时，会将字符串常量加载到内存中，然后执行
    - 字符串常量的内存空间在程序运行时会被操作系统自动分配，用户不需要手动释放
    - 字符串常量的内存空间在程序运行时会被操作系统自动释放，用户不需要手动释放
