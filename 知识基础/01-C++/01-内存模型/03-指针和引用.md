## 指针和引用
1. 指针
    - 定义：指针是一个特殊的变量，它里面存储的的数值为内存里的一个地址，通过*访问内存地址所指向的值

2. 引用
    - 定义：引用不是新定义一个变量，而是给已存在变量取了一个别名，编译器不会为引用变量开辟内存空间，它和它引用的变量共用同一块内存空间。
    - 特性：
        - 引用必须初始化
        - 一个变量可以有多个引用
        - 引用一旦引用了一个实体就不能引用另一个实体了(再次引用其实是重新赋值)
        ```
        int a = 10;
        int& ra = a;
        // 此时ra和a指向同一块内存空间
        int b = 20;
        ra = b;
        // 此时ra还是指向a的地址，ra和a的值都被修改为b的值20
        ```
        - 常引用：常引用不能被重新赋值，也不能被修改
        ```
        int main()
        {
	        const int a = 10;
	        //int& ra = a;    //此时编译报错，因为a是常量，不能通过普通引用来引用a
	        const int& ra = a;
 
	        //int& ra=10;     //编译也会报错，因为10为常量
	        const int& b = 10;
        }
        ```

        - 不同类型的常引用：
        ```
        double d = 23.8;
        // int& rd = d;    //编译报错，因为d是double类型的，不能通过int类型的引用用来引用d
        const int& rd = d; // 编译通过，但是rd并不是指向d的地址
        // d和rd并非是同一块内存 空间，修改d的值并不影响rd，为什么会这样呢？因为const int& rd=d;执行时候，编译器发现double和int之间可以发生隐式转换，于是重新创建了一块整形的空间，将d变量中整形部分放入到临时空间中，而临时空间又是编译器在栈上开辟的，用户并不知道这块空间的名字和地址，自然空间中的值就修改不了了，即临时空间具有常性。
        ```

3. 对比
    1. 引用概念上定义一个变量的别名，指针存储一个变量地址
    2. 引用在定义时必须初始化，指针没有要求
    3. 引用在初始化时引用一个实体后，就不能再引用其他实体，而指针可以在任何时候指向任何一个同类型 实体
    4. 没有NULL引用，但有NULL指针
    5. 在sizeof中含义不同：引用结果为引用类型的大小，但指针始终是地址空间所占字节个数
    6. 引用自加即引用的实体增加1，指针自加即指针向后偏移一个类型的大小
    7. 指针有多级指针，但是引用没有多级引用
    8. 访问实体方式不同，指针需要显式解引用，引用编译器自己处理
    9. 引用比指针使用起来相对更安全
